# Imara — Economics Agent: research cycle &amp; incorporation plan
*Goal: a macro-economics agent that reads the macro environment and translates it into effects on a specific SME's "internal economy" (its statements, ratios, Imara Score and risk). Designed to fit Imara's deterministic / anti-hallucination DNA: real data + deterministic sensitivity maths, LLM for narrative only.*

## 1. Research synthesis (two rounds)
**Macro → firm transmission.** Credit/financial-risk practice links performance to a handful of macro variables — **GDP growth, interest rates, inflation, unemployment, FX** — via estimated **sensitivities/elasticities**; GDP shocks move default rates most, and **SMEs are the most exposed segment** ([IMF stress-testing](https://www.elibrary.imf.org/display/book/9781484368589/ch031.xml), [Moody's](https://www.moodys.com/web/en/us/insights/resources/11-15-06-The-role-of-stress-testing-in-credit-risk-management.pdf)). For a *single firm*, the method is **bottom-up sensitivity**: read exposures off the firm's own statements — floating-rate debt → rate sensitivity; net foreign position × ΔFX × (1−tax) → currency sensitivity; cost structure vs pricing power → inflation sensitivity — turning "macro volatility into a concrete risk-exposure profile" ([CFI](https://corporatefinanceinstitute.com/resources/fixed-income/interest-rate-sensitivity/)).
**Forward-looking scenarios.** IFRS 9 uses **multiple probability-weighted scenarios** (commonly base 50% / upside 25% / downside 25%) because a single scenario understates loss under non-linearity, plus **model overlays** (expert/post-model adjustments) for thin-data SME portfolios ([Moody's ECL](https://www.moodys.com/web/en/us/insights/banking/probability-weighted-outcomes-under-ifrs-9.html)).
**SA data is free &amp; programmatic.** [World Bank Indicators API](https://datahelpdesk.worldbank.org/knowledgebase/articles/889392-about-the-indicators-api-documentation) (no key, JSON, `api.worldbank.org/v2/country/ZA/indicator/...`), the [SARB Web API](https://custom.resbank.co.za/SarbWebApi/) (repo rate, financial stats), and [SAMADB](https://mirror.linux.duke.edu/pub/cran/web/packages/samadb/index.html) (~10k SA series, Python package).
**SA-specific shocks matter.** Electricity tariffs are the standout: **+12.74% (Apr 2025), ~+190% since 2014** — far above inflation; **load-shedding has eased** (≈12 days in 8 months of 2025 vs 83 in 2024), so tariffs &gt; outages now. Inflation ≈5% (SARB target 3–6%); GDP ≈1.3–1.4% ([OECD](https://www.oecd.org/en/publications/oecd-economic-surveys-south-africa-2025_7e6a132a-en/full-report/reforming-south-africa-s-electricity-sector_05fdccb6.html), [Daily Maverick](https://www.dailymaverick.co.za/article/2025-12-26-2025s-business-roller-coaster-from-load-load-shedding-relief-to-budget-chaos-and/)).

## 2. Questions → answers
- **Q1 How to transmit macro to one firm?** Deterministic **bottom-up sensitivity engine** off the firm's statements (rate, inflation, FX, energy). No LLM in the numbers — same pattern as the financial-ratio engine.
- **Q2 Where does grounded macro data come from?** A **curated, dated SA macro snapshot** (repo/prime, CPI, GDP + forecast, ZAR/USD, electricity-tariff path, unemployment) with citations + `as_of` — the same pattern as the SA-legislation corpus — with an **optional live refresh** from the World Bank/SARB API and graceful fallback. Avoids a hard runtime dependency mid-pipeline while staying current.
- **Q3 How does it connect to Imara's outputs?** Three touchpoints: (a) **grounded findings** (macro context tied to the firm's numbers); (b) **macro stress scenarios in the Action Simulator** (base/upside/downside, probability-weighted, IFRS-9 style); (c) a **forward-looking overlay** on the Score — a documented "Macro Resilience" signal / confidence note, NOT a hard score rewrite (avoids double-counting and keeps the Score stable).
- **Q4 Where in the pipeline?** Runs in the **concurrent pre-synthesis tail** (alongside market deep-dive / SA tax / SA legal via `run_independent_agents`): reads prior findings read-only, writes disjoint `macro_*` fields. Deterministic numbers, LLM narrative, cited indicators.

## 3. Incorporation design (how to best build it)
1. **Macro data layer** — `services/macro_data.py`: a curated, dated `SA_MACRO` snapshot (each indicator: value, as_of, source, short trend) + `refresh_from_worldbank()` optional live fetch (no key) with fallback to the snapshot. Mirrors the `sa_knowledge` corpus pattern.
2. **Firm macro-sensitivity engine** — deterministic functions over the existing `financial_figures`/ratios: rate sensitivity (floating-debt × Δrepo → Δinterest → Δinterest-cover); inflation/cost sensitivity (COGS+opex vs gross buffer); FX sensitivity (net foreign position × ΔZAR × (1−tax), gated on an import/export intake flag); energy sensitivity (opex/energy intensity × tariff path). Returns a per-driver "exposure profile" (low/med/high + ZAR impact).
3. **EconomicsAgent** — new `BaseAgent` subclass: prompt grounded with the macro snapshot + the firm's computed sensitivities; emits `AgentFinding`s that cite real indicators ("repo at X%, ~Y% of debt floating → a +100bps hike adds ≈R___ interest, cutting interest cover A→B"). LLM narrates; numbers come from the engine.
4. **Action Simulator integration** — add macro **stress drivers** (repo +/−bps, inflation +pp, ZAR +/−%, tariff +%) and a **probability-weighted 3-scenario macro view** reusing the existing `apply_actions`/Monte-Carlo machinery → "how resilient is this firm to a rate/inflation/tariff shock?"
5. **Score + reporting** — a `macro_resilience` signal + a forward-looking note in the Methodology panel / model card (and the versioned Score contract). Conservative: overlay/flag, not a hard component, unless later calibrated.
6. **Placement** — register in `run_independent_agents` (Phase 2 tail); `macro_*` fields on SharedMemory; surfaced in the dashboard ("Economic Environment" section) + PDF/HTML.

## 4. Build phasing
- **Phase 1 (deterministic core, no API risk):** macro snapshot + sensitivity engine + EconomicsAgent (uses the snapshot) + findings + a dashboard panel. Fully testable; this alone delivers the feature.
- **Phase 2:** macro stress scenarios in the Action Simulator (reuse engine).
- **Phase 3:** optional live `refresh_from_worldbank()`/SARB with caching + `as_of` display.
- **Phase 4 (needs data):** calibrate macro→outcome sensitivities against real results (ties into the deferred Score-calibration work).

## 5. Honest notes
- **Don't double-count:** macro stays an overlay/context + simulator scenario; it should not silently move the headline Score until calibrated.
- **Freshness &amp; provenance:** every macro figure carries an `as_of` + source; stale snapshots must be visibly dated (trust > false precision).
- **SA realism:** include the electricity-tariff trajectory and (eased) load-shedding as first-class SA shocks, not just textbook GDP/rates/FX.
- **Keep the DNA:** all impact numbers deterministic; the LLM only explains. Consistent with the ratio engine, faithfulness check, and finding self-critique.
